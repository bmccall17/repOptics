# 🧠 repOptics Brain

## 1. Identity & Vision
**Project Name:** repOptics (Repo Optics)
**Mission:** To provide clear "optics" and visibility into the entire architecture, function, and interconnected pieces of an application.
**Core Function:** It serves as a tool to give a complete vision of how all pieces of a built application are wired together. It helps developers start off with the *right* architecture from day one and consider all aspects of application "galvanization" before fully executing a project (e.g., discerning gates, requirements, and structural integrity).

## 2. Target Audience
- **Primary Audience:** Initially, the solo developer/creator (you). 
- **Secondary Audience:** Future developers who need a cohesive map of how their software systems and architectures are wired.

## 3. Key Capabilities & Features
repOptics acts as a constant architectural guardian and advisor. Its primary capabilities include:

- **Code Hygiene & Orphan Management:** Detecting and highlighting when code becomes orphaned, unused, or needs to be cleaned up.
- **Dependency & Architecture Mapping:** Showing how the various parts of the codebase are wired together (not necessarily real-time application state, but architectural connectivity).
- **Tooling & Practice Recommendations:** repOptics actively collects tools and best practices. It will proactively act as an advisor, reminding you: *"Hey, there's a tool here [like Snyk] that you haven't installed yet. Consider doing that next."*
- **Galvanization Checklist:** Ensuring that the project meets necessary "gates" and requirements prior to major implementation phases.

## 4. Key Integrations & Practices
- **Security & Vulnerability Management:** Integration and strict adherence to tools like **Snyk** for finding and fixing code vulnerabilities (as mandated by your project rules). repOptics should track the implementation state of tools like this.
